# Changelog — 29 апреля 2026

## Колонки — цвет, редактирование, удаление, drag-and-drop

### Цвет колонки

- Добавлена цветная полоска (`h-1.5`) в верхней части карточки колонки.
- Палитра из 7 дефолтных цветов: slate, indigo, sky, emerald, amber, rose, violet — вынесена в `entities/board/columnColors.ts`.
- Поддержка кастомного цвета через нативный `<input type="color">`.
- Кнопка «No color» сбрасывает цвет в `null`.

### Создание колонки

- В форме `+ Add column` под инпутом появляется палитра: X (без цвета), 7 пресетов, кнопка «+» для кастомного.
- Превью выбранного цвета отображается прямо на карточке формы в реальном времени.
- Форма создания больше не растягивается на всю высоту — `self-start`, прижата по содержимому.
- Кнопка `+ Add column` в состоянии покоя выглядит как компактная кнопка с пунктирной рамкой и иконкой `+`, а не как пустая карточка колонки.

### Редактирование существующей колонки

- При наведении на хедер колонки появляется иконка-карандаш (аналогично редактированию досок в сайдбаре).
- По клику — модалка `EditColumnModal` с:
  - Инпутом названия.
  - Полной палитрой (пресеты + кастомный цвет + сброс).
  - Превью результата (полоска + название).
  - Кнопками Save / Cancel.
- Изменения применяются оптимистично — сразу в UI, с роллбэком при ошибке API.

### Удаление колонки

- В модалке редактирования добавлена кнопка-корзина.
- По клику — инлайн-подтверждение внутри той же модалки: текст с предупреждением + красная кнопка Delete / Cancel. Без второй модалки поверх.
- Удаление оптимистичное с роллбэком.

### Drag-and-drop сортировка колонок

- Колонки можно перетаскивать мышью для изменения порядка.
- Логика — **swap**: как только перетаскиваемая колонка оказывается над другой, они мгновенно меняются местами прямо в UI. Сам обмен и есть визуальная обратная связь — без колец, линий и плейсхолдеров.
- Перетаскиваемая колонка становится полупрозрачной (`opacity-40`), курсор меняется на `grab`.
- Порядок фиксируется в API при drop; при отмене (Escape / бросить за пределами) — откат к исходному состоянию.
- Используется тип `application/column-id` в `dataTransfer` — не конфликтует с drag задач внутри колонок.
- Исправлен баг: `dragstart` от TaskCard всплывал к обёртке колонки и загрязнял drag-данные задачи, ломая оба вида перетаскивания.
- Исправлен баг на бэкенде: `upsert` в эндпоинте reorder падал с 500, потому что Postgres проверяет NOT NULL до `ON CONFLICT` — заменено на параллельные `UPDATE` запросы.

### Права доступа

- Карандаш, кнопка удаления и `+ Add column` видны только пользователям с ролью `owner` или `admin`.
- Роль определяется из `workspace.role` (из `workspace_members`) — не из `board_members`. Исправлена ошибка, при которой owner workspace не мог редактировать колонки.
- Backend: все мутации колонок (`POST`, `PATCH`, `DELETE`, reorder) проверяют роль и возвращают `403` для обычных `member`.

### Backend — новые эндпоинты

- `PATCH /api/v1/columns/:id` — обновление названия и/или цвета колонки.
- `DELETE /api/v1/columns/:id` — удаление колонки.
- `POST /api/v1/boards/:id/columns/reorder` — приём массива `columnIds`, атомарный `upsert` позиций через Supabase.

### База данных

- Миграция: `ALTER TABLE columns ADD COLUMN color varchar(7) NULL`.
- Снапшот схемы до миграции сохранён в `databases/2026-04-29_before_add_color_to_columns.sql`.
- Введено правило: перед каждой миграцией сохранять дамп схемы в `databases/` с датой в имени файла.
