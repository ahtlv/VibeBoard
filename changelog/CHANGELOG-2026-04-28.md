# Changelog — 28 апреля 2026

## Boards & Modals

### Управление досками в сайдбаре

- **Редактирование доски** — при наведении на доску в сайдбаре появляется иконка карандаша. По клику — модальное окно с полями названия и описания, кнопками Save / Cancel / Delete board.
- **Создание доски** — форма создания теперь открывается по центру экрана (модалка). Содержит поля: название (обязательное) и описание (опциональное).
- **Удаление доски** — вместо браузерного `confirm()` реализована стилизованная модалка с иконкой предупреждения (треугольник), явным описанием действия и красной кнопкой подтверждения. После удаления активной доски — автоматический переход на первую оставшуюся.
- **Обновление названия/описания** — после сохранения список в сайдбаре обновляется сразу.

### Backend — новые эндпоинты

- `GET /api/v1/boards/:id` — возвращает полную доску с колонками и задачами (ранее отсутствовал).
- `PATCH /api/v1/boards/:id` — обновление названия и описания доски с проверкой членства в workspace.
- `DELETE /api/v1/boards/:id` — soft-archive через `is_archived = true` (безопасно для FK на колонки/задачи).

### Данные из базы Supabase

- `DashboardPage` больше не создаёт доску с пустыми колонками (`summaryToBoard` удалена). При выборе доски происходит реальный запрос к API — загружаются колонки и задачи из Supabase.
- При переключении между досками показывается skeleton-загрузчик.
- Добавлен маппер `mapBoardDetail` для конвертации snake_case ответа бэкенда в frontend-типы.
- Починен маппер `createColumn` — `board_id` теперь корректно конвертируется в `boardId`.

### Заголовок BoardHeader

- Убрана кнопка `+ Add task` из хедера (она дублировала кнопку внутри колонок).
- Вместо названия workspace под заголовком доски теперь выводится **описание доски** (скрывается, если описание пустое).

### Колонки — создание

- Кнопка `+ Add column` перенесена из хедера в конец ряда колонок.
- В состоянии покоя выглядит как карточка колонки (тот же бордер, фон, шрифт). Без пунктирной обводки.
- По клику — инлайн-форма с инпутом прямо на месте кнопки (Enter — подтвердить, Escape — отмена).
- Создание колонки оптимистичное: колонка появляется сразу, при ошибке откатывается.

### Модалки — центрирование и blur

- Все модалки (создание доски, редактирование, подтверждение удаления) рендерятся через **React Portal** в `document.body`.
- Причина: сайдбар имеет `transition-transform`, что создаёт новый stacking context и ломает `fixed`-позиционирование вложенных элементов.
- Оверлей начинается с `top-14` (под хедером) — хедер остаётся чётким, контент под ним блюрится (`backdrop-blur-sm`).
