# Refactoring Rules

Принципы безопасного рефакторинга. Обязательны для всех задач, помеченных как рефакторинг.

## Two Hats (Martin Fowler)

Рефакторинг и новая фича — разные коммиты, лучше разные PR. Никогда не смешивать в одном изменении «поведение изменилось» и «структура изменилась». Если заметил улучшение по ходу фичи — фиксируй в TODO, делай отдельно.

## Preserve Behavior

Рефакторинг не меняет публичное поведение. Тесты должны быть зелёными до и после без изменений тестов. Если тест пришлось менять — это уже не рефакторинг.

## Small Reversible Steps (Lampson)

Предпочитать обратимые изменения необратимым. Каждый шаг — мелкий и сам по себе корректный. Не укладывать в один коммит несколько несвязанных переездов.

## Strangler Fig для рискованного переезда

Если переезд рискованный (переименование модуля, смена структуры папок, изменение публичного API):
1. Создать новое рядом со старым.
2. Постепенно переводить вызывающий код на новое.
3. Удалить старое только когда на него нет ни одной ссылки.

Не удалять старое превентивно «а вдруг мы перешли».

## Запрет bulk-переименований без аудита

Перед переименованием файла, папки или публичного символа — обязательный `grep -r` по всему репо:
- `.github/`
- `wrangler.toml`, `Dockerfile`, `docker-compose.yml`
- `package.json` (scripts, paths)
- все `*.md`, `CLAUDE.md`
- миграции в `databases/`

Если нашлась хоть одна ссылка — переименование требует явного плана миграции.

## Boy Scout Rule — локально

Улучшать только в радиусе текущей задачи. Попутный рефакторинг соседнего кода запрещён. Если заметил проблему рядом — упомяни пользователю, но не правь.

## Documentation Sync — в том же коммите

При любом изменении стека, структуры папок или публичного API — синхронно обновлять:
- `CLAUDE.md` (корневой)
- `frontend/CLAUDE.md`
- `backend-ts/CLAUDE.md`

Documentation drift — баг, не технический долг. Несинхронная документация хуже отсутствующей.

## Если есть сомнения — не трогай старое

Создать новое по правилам. Старое оставить нетронутым. Миграция — отдельная задача с явным планом.
