# Changelog

После каждого крупного пуша в GitHub — писать changelog в папку `changelog/`.

## Формат имени файла

```
changelog/CHANGELOG-YYYY-MM-DD-<тема>.md
```

Примеры:
- `changelog/CHANGELOG-2026-04-29-columns.md`
- `changelog/CHANGELOG-2026-04-30-task-modal.md`

## Что считается крупным пушем

Новая фича, значимое изменение UX, новый эндпоинт, миграция БД, рефакторинг с последствиями — всё, что меняет поведение продукта.

## Мелкие правки — аккумулировать

Косметические фиксы, баги, hover-состояния, пикселепиксельные правки — **не писать отдельный changelog**. Копить в голове до следующего крупного пуша, затем включить одним пунктом («Мелкие правки и баги»).

## Структура файла

```markdown
# Changelog — DD месяц YYYY

## Название фичи или области

### Подраздел (если нужен)

- Что добавлено / изменено / удалено
- Что изменилось в UX
- Новые эндпоинты бэкенда
- Изменения в БД
```

Писать с точки зрения **продукта**, не кода. Что изменилось для пользователя — в первую очередь.
