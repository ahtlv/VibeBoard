# DB Migrations

Перед каждой миграцией базы данных — сохранять дамп текущего состояния схемы в папку `databases/` в корне проекта.

## Формат имени файла

```
databases/YYYY-MM-DD_<описание_изменения>.sql
```

Примеры:
- `databases/2026-04-29_before_add_color_to_columns.sql`
- `databases/2026-04-30_before_add_labels_table.sql`

## Цель

При откате к любому git-коммиту можно точно восстановить соответствующее состояние базы: находишь ближайший дамп по дате перед коммитом — и откатываешь базу к нему.

## Что сохранять

Полный дамп схемы (структура таблиц, индексы, политики RLS, триггеры) — без данных. Используй Supabase MCP (`execute_sql` с запросами к `information_schema` и `pg_catalog`) или любой другой доступный способ получить DDL.

## Порядок действий

1. Сохранить дамп в `databases/YYYY-MM-DD_before_<migration_name>.sql`
2. Применить миграцию через `apply_migration`
3. Зафиксировать в git вместе с изменениями кода
